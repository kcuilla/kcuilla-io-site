I am an avid R-user. It is my tool of choice when it comes to nearly any project that involves data analysis and visualization. The thing I enjoy most about R is the community. I’ve learned a ton of knowledge from people in the community who constantly share their work and help others. Lately, I’ve been trying to figure out the best way that I can share the things I’ve learned in order to give back to the community, and I stumbled upon the blogdown package.

In this post, I will give a brief walk-through of how I was able to create this website using blogdown and a few other tools, and also share some tips and resources that were beneficial to me during development.

Getting Started

I highly recommend reading the first few chapters of blogdown: Creating Websites with R Markdown by Yihui Xie, Amber Thomas, and Alison Presmanes Hill. Not only does the book teach you how to use the blogdown package in R, but it also provides a good tutorial on Hugo, which controls the theme of your website, and ways to deploy your website such as through Netlify, which is what I’m using to host my site.

Github and RStudio

The first thing you will need to do is create a new repository on Github. If you are new to Github and are not sure how to do this, I recommend reading this how-to article first.

Once you’ve created the repository in Github, open RStudio and create a new project with Git version control by File -> New Project -> Version Control -> Git

  • Repository URL: copy and paste the URL of the Github repository you just made. For me, this is https://github.com/kcuilla/kcuilla.github.io
  • Project directory name: copy and paste the name of your Github repository. For me, this is kcuilla.github.io
  • Create project as a subdirectory of: this is the folder location on your local computer where all of your files will be stored. It doesn’t need to be named anything specific, you may name it anything you like.

Blogdown and Hugo

Install the blogdown package and Hugo in RStudio:

install.packages("blogdown")
library(blogdown)
blogdown::install_hugo()

Pick a theme for your website from Hugo Themes. The theme I settled on was the Academic theme because I liked both the look and the functionality, but there are plenty of other options available as well.

Once you have chosen a theme, you can download it using the following blogdown command:

# Replace user/repo from the theme's Github repository
blogdown::new_site(theme = 'user/repo')

The files that control the contents of the theme should now be available in your working directory and you can view them by navigating through the Files tab in RStudio as shown below:

(Note: I will not go into detail on how to edit the configuration files as these differ for each theme. However, there are normally tutorials provided by the theme creators, and if you’re interested in the edits that I made for the Academic theme, you can view them on my github).

In order view your configuration edits to your site, run blogdown::serve_site() and you will be able to see a preview of your website in the Viewer tab in RStudio (or you have the option to view it in your browser by clicking the ‘Show in new window’ icon):
